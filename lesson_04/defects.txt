Баг-репорт №1
Заголовок: StringUtils.capitalize() поднимает AttributeError вместо TypeError для нестроковых аргументов.
Описание: При передаче нестрокового значения метод capitalize() падает с AttributeError, хотя должен явно поднять TypeError.
Шаги: 
import pytest
from string_utils import StringUtils
@pytest.mark.parametrize(
    "input_text, expected_output",
    [
        ("", ""),
        ("123abc", "123abc"),
    ],
)
def test_string_utils_capitalize_valid(input_text, expected_output):
    utils = StringUtils()
    assert utils.capitalize(input_text) == expected_output


@pytest.mark.parametrize("invalid", [None, 123])
def test_string_utils_capitalize_invalid(invalid):
    with pytest.raises(TypeError):
        StringUtils().capitalize(invalid)
Ожидаемый результат:
TypeError: Expected str, got NoneType
Фактический результат:
AttributeError: 'NoneType' object has no attribute 'capitalize'
AttributeError: 'int' object has no attribute 'capitalize'
Окружение: 
Python 3.14.0
pytest 9.0.1
Серьезность: S2
Приоритет: P1
